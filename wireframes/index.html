A

Track A — Akses / Setup

Dari pendaftaran sampai bisa masuk app. Sekali jalan per tenant/user.

  1. 1 Register register.html Daftar tenant publik — data tenant + pilih vertical/usage, lalu submit.
  2. 2 Pending pending.html Tenant berstatus pending — menunggu superadmin mengaktifkan.
  3. 3 Superadmin activate superadmin-users.html Console superadmin — aktifkan (masa aktif + kuota) / suspend / buat akun.
  4. 4 Onboarding onboarding.html Konfirmasi vertical → set modul + entitlements aktif; buat workspace produk pertama.
  5. ↪ Login · entry terpisah, user existing login.html Masuk untuk user lama → diarahkan sesuai status (Pending / Onboarding / Dashboard).
Register → Pending → Superadmin activate → Onboarding → masuk Track B (Login = pintu masuk terpisah untuk user existing)
B

Track B — Produk (core)

Alur kerja harian di dalam app shell (sidebar + topbar white-label). 1 workspace = 1 produk.

  1. 1Dashboard dashboard-shell.html App shell + KPI ringkas (lead, obrolan, deal, kuota AI). Entry point modul aktif.
  2. 2Workspace workspace.html Hub 1 produk: funnel, kontak ter-akuisisi (badge B2C/B2B), sales script, eksekusi obrolan.
  3. 3Enrichment / Discovery enrichment.html Cari lead (discovery) → enrich profil → klasifikasi B2C/B2B. Surface mandiri + tombol di Workspace/Contacts.
  4. 4Contacts (CRM) contacts.html Daftar kontak/lead tersegmentasi B2C/B2B (tab + filter) + drawer detail/edit. CRM first-class.
  5. 5Inbox (WA) inbox.html Percakapan WhatsApp — list chat + thread + panel konteks lead + AI assist obrolan konsultatif.
  6. 6Pipeline pipeline.html Board kanban deal per tahap + drawer deal. Atribusi per akun sales rep.
Dashboard → Workspace → Enrichment/Discovery → Contacts (B2C/B2B) → Inbox (WA) → Pipeline (loop harian: cari & enrich lead → simpan kontak → obrol di WA → gerakkan deal)
⚙

Pengaturan

Konfigurasi. Branding = grain USER; pengaturan lain (AI, tim, billing, dst) = grain TENANT.

  1. B Branding · per-user branding.html Ubah seluruh token warna + logo + favicon + Custom CSS. Default Coral Sunset; reset + live preview. Hanya untuk user itu.
  2. … Pengaturan lain · belum di-wireframe — AI/provider · Tim & peran · Billing/kuota · Mailboxes · WA gateway · Extension · Knowledge base · Handoff · Compliance · Diagnostics. (Lihat inventaris di bawah.)
